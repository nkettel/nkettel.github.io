Tank Battle Game

Python (CMU Graphics)

A 4-player competitive tank battle game inspired by Arras.io. Players control tanks with different classes, collect power-ups, and compete for the highest score. Built with CMU Graphics library featuring real-time game physics and collision detection.

Key Features:

  • 4-player simultaneous gameplay with unique control schemes for each player
  • Multiple tank classes (Basic, Pounder, Sniper, Repeater) with different stats
  • Dynamic polygon spawning system with different shapes and scoring values
  • Real-time collision detection and damage system
  • Score tracking with upgrade unlocks at 250+ points
  • Polygon rotation and health regeneration mechanics

Challenges & Solutions:

Managing collision detection between multiple game objects (tanks, bullets, polygons) was complex. I solved this by creating a modular collision system that checks object types and applies appropriate damage. Balancing tank statistics was also crucial—each tank type needed distinct advantages without being overpowered.

Lessons Learned:

This project taught me the importance of game state management and efficient object tracking. I learned how to structure code for multiple concurrent game entities and the mathematical concepts behind angle calculations and distance-based collision detection.

Controls:
Blue (P1): W/A/S/D move, E/R rotate, Q shoot
Red (P2): Y/G/H/J move, U/I rotate, T shoot
Green (P3): P/L/;/' move, [/] rotate, O shoot
Pink (P4): Arrow keys move, ./​/ rotate, Enter shoot

My Development Journey Through Projects

These five projects represent my evolution as a programmer. Starting with game development using Python and CMU Graphics, progressing through foundational Java skills for desktop applications, and advancing to web technologies with JavaScript and React. Each project has taught me valuable lessons about software architecture, problem-solving, and user experience design.

What these projects taught me:

  • Game Physics & Collision Detection: Understanding coordinate systems and real-time collision detection
  • Architecture Matters: Good design decisions early prevent technical debt later
  • User Experience is Critical: The best code means nothing if users can't or won't use it
  • Testing is Essential: Building tests as you develop catches bugs early and builds confidence
  • Documentation Saves Time: Well-documented code is significantly easier to maintain and enhance
  • Debugging is a Skill: Learning effective debugging techniques is as important as learning languages
  • Collaboration Improves Code: Code reviews and peer feedback lead to better solutions
  • Performance Matters: Writing code is just the first step; optimization and efficiency are crucial
